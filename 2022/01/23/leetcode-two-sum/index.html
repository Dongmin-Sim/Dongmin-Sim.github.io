두 수의 합

[리트코드] 1. 두 수의 합 https://leetcode.com/problems/


박상길님의 파이썬 알고리즘 인터뷰 책을 참고하여 풀이하였음을 미리 밝힙니다.

문제

정수 숫자들의 배열과 정수 targe이 주어질 때, 더해서 target이 되는 두개의 숫자의 인덱스 값을 리턴하시오.

조건 : 각각의 입력값은 오직 하나의 케이스 정답이 존재하며, 같은 원소를 두 번 사용해서는 안된다.


예시

Input : nums = [2,7,11,15], target = 9
Output : [0,1]


핵심 아이디어

Brute-force

첫 번째로 생각할 수 있는 가장 쉬운 접근 방법은 brute-force로 모든 조합을 다 더해보는 것이다. 이는 이중 반복문으로 해결할 수 있으며 시간 복잡도는 전체 요소를 두번 반복해야 하므로 $O(n^2)$가 소요된다. 코드는 다음과 같다.

In 연산자

조금만 생각을 바꾸어 보면 정답이 될 수 있는 케이스는 오직 하나이므로, target값에서 현재 값을 뺀 값이 nums 리스트에 존재하게 된다. 그러므로 다음과 같은 순서로 문제를 풀이할 수 있다.

  1. target에서 현재 값을 뺀다.
  2. 리스트에서 뺀 값의 index를 찾는다.
  3. 현재 값의 인덱스와, 뺀 값의 인덱스를 리턴한다.

코드는 다음과 같다.

여기서 주의해야할 점은 target - current 값을 current의 인덱스 이후에서 찾아야한다는 점이다.
따라서 if문에서 i번째 다음부터 슬라이싱하여 in을 수행해야 한다.
한 가지 더 주의해야할 점은 target - current의 인덱스 값은 슬라이싱된 리스트에서 찾아진 인덱스 값이므로 사실상 슬라이싱된 리스트는 i+1번째부터 시작한다는 점을 명심해야 한다.

이 풀이는 시간복잡도가 O(n)인 것 같지만 사실 in연산도 O(n)이므로 사실상 전체 시간복잡도는 O(n^2)이다. 하지만 Brute-force 방법보다는 내부에서 구현된 in연산이 훨씬 빠르다.

해시테이블

이 방법은 미처 생각하지 못했던 방법이다. 위의 코드들도 통과하는데는 무리가 없지만 다음과 같은 방법을 사용하면 시간복잡도를 크게 줄일 수 있다.
위의 방법들은 모두 O(n^2)으로 풀이가 했지만, 이를 O(n)으로 풀이하는 것이 가능하다.
딕셔너리를 사용하여 target - current의 값을 키로, 해당하는 인덱스 값을 키의 값으로 저장한다면 즉시 조회가 가능하다. 딕셔너리는 해시 테이블로 구현되어있고, 이는 조회에 O(1)에 가능하다.

  1. 딕셔너리에 current - index의 쌍으로 저장한다.
  2. 반복문을 둘면서 target - current의 값이 딕셔너리에 있는지 확인한다.
  3. 만약 딕셔너리안에 존재하고, 해당 key의 값이 현재 인덱스와 일치하지 않는다면(조건 : 같은 원소는 2번 사용하지 않는다.)
  4. 현재 값의 인덱스와, 뺀 값의 인덱스를 리턴한다.

코드는 다음과 같다.

개인적으로 한 가지 의문점이 들었다. 위에서 in연산은 O(n)의 시간복잡도를 가진다고 했다. if문에서 in연산이 수행되고 있는데, 그렇다면 시간 복잡도는 O(n^2)이 아닌가?

답은 “그렇지 않다”고 한다. 시간복잡도를 나타내는 Big-O 표기법은 입력값에 대한 알고리즘 수행 시간을 설명하는 계산복잡도를 의미하는데 이는 절대적인 성능 수치가 아니다. 때문에 if문은 입력값 n에 의해 실행시간이 증가하는 비율에는 영향을 줄만큼 크지 않을 수 있기 때문에(시간복잡도에 영향을 주는 요소에 더 집중한다고 이해) big-o 표기에는 영향을 미치지 않는다는 것 같다. 하지만 실제 프로그램의 성능에는 유의미한 차이가 있을 수 있다고 한다. 참고한 글은 다음과 같다.
