프린터

[프로그래머스] 스택/큐 프린터 https://programmers.co.kr/learn/courses/30/lessons/42587


프로그래머스 코딩테스트 연습문제 중 스택/큐 카테고리의 프린터 문제입니다. 문제 내용은 실제 문제와 다를 수 있으니 상단 링크를 참조하세요.

문제

중요도가 높은 문서를 먼저 인쇄하는 프린터가 존재합니다. 이 프린터는 아래와 같은 방식으로 인쇄를 수행합니다.

  1. 인쇄 대기목록의 가장 앞에 있는 문서(J)를 대기목록에서 꺼냅니다.
  2. 나머지 인쇄 대기목록에서 J보다 중요도가 높은 문서가 한 개라도 존재하면 J를 대기목록의 가장 마지막에 넣습니다.
  3. 그렇지 않으면 J를 인쇄합니다.

대기 목록의 문서 중요도가 순서대로 담긴 배열 priorities, 내가 인쇄를 요청한 문서의 index 위치인 location이 매개변수로 주어진다. 이때, 내가 요청한 문서가 몇 번째로 인쇄되는지 return하시오.

예시

input : priorities-[2, 1, 3, 2], location-2
output : 1

인쇄를 요청한 문서의 location은 2, 3의 중요도를 가진 문서의 인쇄 순서를 구하면 된다.

  • location 0(2)은 뒤에 더 중요도가 높은 문서가 있으므로 대기목록 뒤로 위치함.
  • location 1(1)은 뒤에 더 중요도가 높은 문서가 있으므로 대기목록 뒤로 위치함.
  • location 2(3)은 뒤에 더 중요도가 높은 문서가 없으므로 인쇄, 인쇄를 요청한 문서이므로 1을 return

핵심 아이디어

본 문제는 순차적으로 문서를 처리한다는 점에서 큐 자료구조를 이용하는 것이 좋아보인다. 다만, 문서의 순서에 중요도가 존재하며 중요도에 따라 문서를 처리해야 한다는 조건이 존재한다. 따라서 큐에서 나온 문서의 중요도와 아직 큐에 존재하는 문서들의 중요도를 얼마나 잘 효율적으로 비교하느냐까지 고려해야하는 문제인 것 같다.

또한 동일한 중요도를 가진 문서들이 존재할 때 매개변수로 주어지는 location 정보를 고려해여 해당 location의 문서의 순서를 정확히 구해야하는 문제도 존재한다.

그래서 다음과 같은 흐름으로 풀이하였다.

  • 큐 생성
  • 큐 안에 priorities의 원소들을 (index, element)의 튜플로 저장
  • 큐의 첫번째 요소를 pop한 요소와 남은 큐의 요소들의 값을 비교
    • 만약 pop한 값 뒤에 더 중요한 문서가 있다면 다시 큐에 넣음
    • 만약 pop한 값의 중요도가 제일 높으면 카운트+1
      • 만약 pop한 값의 index가 location과 동일하다면 카운트 리턴
  • 위의 과정을 반복

큐

큐에서 pop된 요소의 중요도를 max()연산을 통해 뒤에 더 중요한 문서가 있는지 검사하도록 풀이하였다. 문제는 통과하지만 python에서 max()연산은 모든 요소를 비교하는 방식으로 수행되기 때문에 $O(N)$의 시간복잡도를 갖는다. 때문에 좋은 방법은 아닌 것 같았다. 다른 사람의 풀이를 보고 이 부분을 개선할 수 있는 방법을 찾았다.

any를 이용한 조건 비교

any()를 사용하여 조건 중 하나라도 참이 있으면 True가 되도록 7번 line을 수정했다. 이 경우에 항상 모든 원소들을 비교하지 않을 수 있기 때문에 원소의 개수가 많을 때, 앞선 풀이보다는 빠를 것으로 예상되었다.

모든 테스트 케이스에서 빠른 것은 아닌 듯 하나 대부분 max()를 이용한 조건 비교의 경우보다 빠른 것 같다.
