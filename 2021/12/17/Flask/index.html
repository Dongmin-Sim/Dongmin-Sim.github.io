ORM이란

ORM이란 Object-Relational-Mapping 객체-관계 매핑을 의미한다. 객체와 관계형 데이터 베이스의 데이터를 매핑하는 것을 의미한다. 데이터 베이스에 객체를 통해 접근하는 방법으로 사용된다.

관계형 데이터베이스의 테이블 - 객체지향 프로그래밍의 클래스

이는 SQL질의어가 없어도 데이터베이스를 다룰 수 있도록 도와준다.

ORM의 장점

  • 테이블 구조가 변경될 때, ORM 모델만 수정하면 된다.
    • 데이터 구조가 각각 객체에 매핑되기 때문에 편리하다.
  • 쿼리를 직관적으로 이해할 수 있다.
    • SQL 쿼리가 아닌 객체 형태로, 코드로 데이터를 조작할 수 있어 훨씬 직관적으로 처리할 수 있다.
    • 객체형태로 작성하기 때문에 코드의 가독성을 올려준다.
    • SQL의 절차지향적인 코드가 아니라 객체 지향적인 코드로 사용이 가능하다.
  • 재사용성, 유지보수가 쉽다.
    • 객체 형태로 작성되므로, 해당 객체를 재사용하거나, 유지보수하기 쉽다.


SQLAlchemy

SQLALchemy란 파이썬 ORM라이브러리로써 파이썬 코드에서 DB와 연결하기 위해 사용할 수 있는 라이브러리이다.

서버가 DB와 연결(connecion)하는 부분부터, DB를 다루는 CRUD, 서버와 분리되는 모든 과정 등을 SQLALchemy로 관리할 수 있다.

DB, Model 생성

​ db = SQLAlchemy() 객체를 만들어 사용함

  • SQLAlchemy()에 있는 model기능을 상속하여 class를 만들어준다.

  • db.Column으로 데이터베이스의 컬럼을 명시해주고, 컬럼의 속성을 지정해줄 수 있다.

  • __tablename__으로 DB table을 명시해줄 수 있다.

  • user이라는 테이블을 다룰 때, User클래스로 접근하여 다룰 수 있다.

    __init__ 을 통해 보다 간단하게 사용할 수도 있다.


user는 파이썬 클래스이며, DB의 user 테이블과 매핑하여 사용함.

DB의 테이블과 매핑되는 클래스를 모델이라고 함.
