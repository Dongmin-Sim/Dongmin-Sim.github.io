파이썬 표준 라이브러리 정리

표준 라이브러리란 자주 사용되는 표준 소스코드를 미리 구현해 놓은 라이브러리를 뜻함.
이보다 더 다양한 라이브러리들이 존재하지만 알고리즘 코딩테스트에서 주로 자주 쓰이는 표준라이브러리들을 정리함.

1. 내장 함수

파이썬 내부에서 별도의 import 없이 사용 가능한 함수들. 코드 작성에서 있어서 필수적인 함수들이 있는데 이러한 함수들을 내장함수(Built-in)라고 함.

input()

Input() 함수는 말 그대로 사용자로부터 입력받는 함수임. 주로 코딩테스트에서는 문제에서 주어지는 변수 값들을 입력받기 위해 사용. input() 으로 받는 값들은 str 의 데이터 형태로 받아지기 때문에 int 형으로 사용해야할 때는 꼭 int() 를 한번 감싸주어야함. 이를 편리하게 해주기 위해 여러개의 변수를 입력받을 때, map 함수를 사용하여 데이터 형 변환을 해주기도 함.

⭐️sort() & sorted()

sort() 와 sorted() 함수 모두 iterable한 데이터들을 정렬하는 기능을 가지고 있지만 사용법이 약간 다름.

  • sort() 는 iterable객체 자체를 정렬하는 기능을 가지고 있음.

    • ```python
      a = [5, 2, 3, 1, 4]

      a.sort()
      print(a) # [1, 2, 3, 4, 5]

이 두 함수는 기본값(reverse=False)으로 오름차순의 정렬기준을 가지고 있으며 reverse=True 속성으로 내림차순으로 정렬하는 것이 가능함.

min() & max()

min()함수는 2개이상의 파라미터를 받아 가장 작은 값을 반환함. 반대로 max()함수는 파라미터가 2개 이상 들어왔을 때 가장 큰 값을 반환함.

abs

abs() 함수는 절댓값을 반환해주는 함수이다. 계산값에서 절대값을 구해야하거나, 어떠한 거리를 구해야할 때 유용하게 사용할 수 있다.

2. Itertools

iterable(반복적인, 순환적인) 자료형의 데이터를 다루기 위한 라이브러리임. 주로 코딩테스트에서 자주 사용되는 클래스는 다음과 같음

permutations

  • iterable 한 객체에서 파라미터 옵션값만큼 원소를 뽑아 일렬로 나열하는 모든 경우를 계산해주는 기능을 가지고 있음(순열)
  • permutations 는 클래스이므로 객체 초기화 이후에는 리스트 자료형으로 변환하여 사용함.

combinations

  • iterable 한 객체에서 파라미터 옵션값만큼 원소를 뽑아 순서를 고려하지 않고 나열하는 모든 경우를 계산해주는 기능을 가지고 있음(조합)
  • combinations 또한 클래스이므로 객체 초기화 이후에는 리스트 자료형으로 변환하여 사용함.

product

  • permutations 와 비슷한 기능이지만, 일렬로 나열하는 모든 경우를 계산해주는 기능(순열) + 중복허용
  • product 객체를 초기화할 때는 뽑고자하는 데이터의 수를 repeat 속성값으로 넣어줌.
  • product 는 클래스이므로 객체 초기화 이후에는 리스트 자료형으로 변환하여 사용함.

combinations_with_replacement

  • combinations 와 비슷한 기능이지만, 순서를 고려하지 않고 나열하는 모든 경우를 계산해주는 기능(조합) + 중복허용
  • combinations_with_replacement 는 클래스이므로 객체 초기화 이후에는 리스트 자료형으로 변환하여 사용함.

itertools 클래스 기능별 비교

중복 ❌ 중복 ✅
순열(나열) permutations product
조합 combinations combinations_with_replacement

3. collections

유용한 자료구조들이 들어있는 라이브러리 주로 코딩테스트에서 deque, Counter 을 가장 많이 사용함.

⭐️deque

파이썬에서는 deque를 사용하여 큐를 구현. deque는 파이썬의 리스트 자료형과 유사하게 원소를 추가하거나, 삭제하는 기능을 가지고 있다. 리스트와 다른점은 맨 앞 또는 맨 뒤의 원소를 추가하거나, 삭제하는데 있어서 리스트는 $O(N)$의 시간복잡도가 소요되지만, deque의 경우 앞의 모든 경우(원소 추가, 삭제)에서 $O(1)$의 시간복잡도를 보장한다는 점에서 차이점을 갖는다. 또 다른 차이점은 deque는 인덱싱이나 슬라이싱의 기능을 제공하지 않는다. 대신에 데이터의 시작 부분과 끝부분에서 원소를 추가하거나, 삭제하는 경우에는 빠른 시간을 보장함으로 스택이나 큐 자료구조를 사용해야할 때 효과적으로 사용될 수 있다.

  • deque 에서 맨 뒤에 원소를 추가할 때는 append(x) 를 사용하여 원소를 추가할 수 있다.
  • deque 에서 맨 앞에 원소를 추가할 때는 appendleft(x) 를 사용하여 원소를 추가할 수 있다.
  • deque 에서 맨 뒤에 원소를 **삭제(반환)**할 때는 pop() 를 사용하여 원소를 삭제(반환)할 수 있다.
  • deque 에서 맨 앞에 원소를 **삭제(반환)**할 때는 popleft() 를 사용하여 원소를 삭제(반환)할 수 있다.

스택이나 큐 자료구조는 하나의 큰 원통형을 생각하면 된다.

  • deque 로 큐를 구현하는 경우에는 원소를 삽입할 때, append(x), 삭제할 때 popleft() 를 사용하므로써 구현가능
  • deque 로 스택을 구현하는 경우에는 원소를 삽입할 때, append(x), 삭제할 때 pop() 를 사용하므로써 구현가능

Counter

Counter 는 횟수를 세어주는 기능을 가지고 있는 함수이다. Iterable 한 객체가 주어졌을 때 지정값이 객체 내에서 몇 번 등장했는지 횟수를 세어준다.

4. bisect

이진 탐색을 쉽게 구현할 수 있도록 파이썬에서 이와 관련된 라이브러리를 제공함. Bisect 라이브러리는 ‘정렬된 배열에서’ 특정한 원소를 찾아야할 때 매우 효과적으로 사용됨. bisect라이브러이에서 bisect_left(), bisect_right() 함수가 가장 자주 사용되며, 시간복잡도 $O(logN)$을 보장한다.

bisect_left(array, x)

정렬된 배열 array에서 x 요소가 들어갈 수 있는 가장 왼쪽 인덱스를 반환해주는 기능이다.

bisect_right(array, x)

정렬된 배열 array에서 x 요소가 들어갈 수 있는 가장 오른쪽 인덱스를 반환해주는 기능이다.

bisect_right 결과값에서 bisect_left 결과값을 빼면 해당 범위안에 해당하는 값의 요소들이 얼마나 있는지 알 수 있음. 이를 이용하여 정렬된 배열에서 특정 범위의 값에 속하는 원소의 개수를 출력하는 사용자 정의 함수를 만들어 사용할 수 있다.

5. heapq

힙(heap) 자료구조를 사용하기 위해 제공되는 라이브러리, heapq는 다익스트라 최단 경로 알고리즘을 포함해 다양한 알고리즘에서 우선순위 큐 기능을 구현하고자 할 때 사용됨. 파이썬의 힙은 최소힙(min heapq)으로 구성되어 있음. 원소를 넣었다 빼는 것만으로 오름차순 정렬이 완료되며 시간복잡도는 $O(NlogN)$을 보장한다.

  • heapq 에서 원소를 추가할 때는 heapq.heqppush(x) 를 사용하여 원소를 추가할 수 있다.
  • heapq 에서 원소를 반환할 때는 heapq.heqppop(x) 를 사용하여 원소를 반환할 수 있다.

파이썬에서 최소힙을 구현하는 방법은 다음과 같음. 파이썬에서는 최대 힙(max heapq)를 제공하지 않기 때문에 추가 및 반환 과정에서 - 음수 부호를 붙여 추가한 후 반환시 다시 음수부호를 붙여 원래의 부호를 갖게 하도록 약간 변형하면 반대로 내림차순 정렬이 가능하다.

6. math

자주 사용되는 수학적인 기능을 포함하고 있는 라이브러리. 팩토리얼, 제곱근, 최대공약수, 상수 등을 제공한다.

sqrt

입력값의 제곱근을 반환함.

factorial

팩토리얼의 값을 반환함 $x!$

pi & e

파이(pi), 자연상수(e)
