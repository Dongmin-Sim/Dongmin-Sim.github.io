IP와 서브넷 마스크

IP와 IP address, IP의 종류, 서브넷 마스킹, 서브넷


IP(Internet Protocal)와 IP 주소(address)

IP란 인터넷 프로토콜의 약자로 데스크톱, 서버, 휴대기기 등 **기기 간 정보(패킷)**를 주고 받을 수 있도록, 통신을 위해 약속된 규약을 의미한다.

실제로 각각의 컴퓨터가 IP 통신을 하기 위해서는 기기의 식별할 수 있는 무언가가 필요한데, 그 무언가를 **IP 주소(address)**라고 한다. 네트워크에 있는 컴퓨터 또는 장치의 식별자라고 할 수 있다. 통상 사용되는 “너 IP가 뭐야”, “컴퓨터 IP 알려줘” 에서의 IP는 IP 주소를 의미한다.

IP는 인터넷 프로토콜, IP주소는 IP통신을 위한 기기 고유 주소

IP 주소는 다시 두 부분으로 나눌 수 있는데, 하나는 네트워크 주소, 하나는 호스트 주소로 이루어져있다. 네트워크 주소는 해당 네크워크의 주소를 나타내고, 호스트 주소는 해당 네트워크 안에서 다시 기기를 식별하기 위한 주소를 의미한다.

왜 이렇게 IP 주소가 구성될까, 해당 이유는 뒤에 나오는 서브넷팅과도 관련이 있다.



IP 주소 체계

IPv4

IPv6



IP의 종류와 NAT

  • Private IP
    • Public IP의 절대적인 개수와 제한을 보완하기 위해 내부에서만 사용되는 IP
  • Public IP
    • 인터넷 상에서 직접적으로 사용되는 고유의 IP

NAT

여러개의 Privat IP를 가진 서버가 인터넷과 통신하기 위해 Public IP로 변환하는 작업을 의미한다. 다른 말로 하면 Public IP를 이용하여 여러개의 Private IP를 가질 수 있고 이러한 Private IP가 인터넷과 통신할 수 있게 해주는 작업을 의미한다.



서브넷팅(Subnetting)과 서브넷 마스크(subnet mask)

서브네팅은 네트워크를 나누는 것, IP를 분할하는 것이다. -> 서브넷 마스크를 일부 변경하여, 논리적으로 네트워크 대역을 나누어 IP를 할당한다.

앞서 IP주소는 네트워크 주소와 호스트 주소로 나뉜다고 했는데, 이를 구별해 주는 것이 서브넷 마스크이다.
